Even with the best EMM you're going to have to do some scripting and repackaging. Because many packages you would deploy don't follow the recommended Apple guidelines. For example we have one application which specifies in its installer that the version is "0.0" which really confuses any package management tool especially when you're pushing new versions, and it doesn't properly declare its app ID so you need to fiddle with the munkiconfig files otherwise it'll end up in an endless install loop. Others need a shell script to install instead of a proper .APP or .PKG...

VPP might be an option for easy mobile-style deployment but only works for apps that are in the app store (most aren't) and then you still need to have a way to send the right settings to it so you'll still have to write some plist XML.

And even built-in profile stuff like FileVault doesn't work with things like AD accounts without significant massaging of account settings

PS: Whitebox packages from Stephane Sudre is a brilliant tool for easily making your own macOS packages. http://s.sudre.free.fr/Software/Packages/about.html And it's free!
